logtwothirds — the sorting barrier, measured

Duan, Mao, Mao, Shu and Yin's O(m log2/3 n) algorithm (STOC 2025) computes shortest paths asymptotically faster than Dijkstra. The theory leaves one question open: does it actually run faster? logtwothirds is a Rust library with Python bindings built to answer that — a faithful implementation verified bit-for-bit against an audited reference (distances and settlement order), an engineered variant family, and a full benchmark record.

1.10–1.20×engineered BMSSP vs optimized Dijkstra at n = 10⁶–10⁷ — the smallest published gap
26–128×the faithful implementation's gap, narrowing with n at the predicted rate
n ≈ 2400 000extrapolated crossover — beyond any physically storable input
520+adversarial graphs in the bit-exact correctness gate

The verdict is a negative result, stated without spin: the asymptotic advantage is real, and no practical input size realizes it. Along the way the study caught an implementation pitfall in the paper's relax-on-equality rule — a duplicate cascade that random float-weighted tests cannot detect and a real road network triggers immediately.
