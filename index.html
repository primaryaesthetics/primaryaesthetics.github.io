Project · 2026

beyond-dijkstra — the successor algorithm, implemented first and audited

In February 2026 Duan, Mao, Shu and Yin sharpened their own sorting-barrier result to O(m √(log n · log log n)). Pure theory, no implementation anywhere. beyond-dijkstra is the first: a Rust port gated bit-for-bit against an audited Python reference, a fifteen-entry audit of the paper's proofs against the running code, and a hand-proved completeness theorem for the repaired engine. The audit found two pseudocode crashes on ordinary graphs and a counterexample to the completeness clause of Lemma 3.7 — the clause that licenses the most standard optimization there is, and a variant that trusts it returns a wrong distance.

1.8–2.2×best engineered variant vs optimized Dijkstra; what remains is Dijkstra carrying the paper's labels
38–182×the faithful implementation's gap across random graphs and a road network
70 verticesthe counterexample graph: a lemma-trusting variant returns 23.59375 for a true 23.546875
n < 2⁴⁰⁹⁶below this size the paper's parameter prescription violates its own validity condition

The algorithm computed exact distances on every input run; one of the proofs that it must does not hold. Both halves of that verdict, and the repair with its proof, rest on the same public verification record.

Project · 2025–2026

logtwothirds — the sorting barrier, measured

Duan, Mao, Mao, Shu and Yin's O(m log^2/3 n) algorithm (STOC 2025) computes shortest paths asymptotically faster than Dijkstra. The theory leaves one question open: does it actually run faster? logtwothirds is a Rust library with Python bindings built to answer that: a faithful implementation verified bit-for-bit against an audited reference (distances and settlement order), an engineered variant family, and a full benchmark record.

1.10–1.20×engineered BMSSP vs optimized Dijkstra at n = 10⁶–10⁷, the smallest published gap
26–128×the faithful implementation's gap, narrowing with n at the predicted rate
n ≈ 2⁴⁰⁰ ⁰⁰⁰extrapolated crossover, beyond any physically storable input
520+adversarial graphs in the bit-exact correctness gate

The verdict is a negative result, stated without spin: the asymptotic advantage is real, and no practical input size realizes it. Along the way the study caught an implementation pitfall in the paper's relax-on-equality rule: a duplicate cascade that random float-weighted tests cannot detect and a real road network triggers immediately.
